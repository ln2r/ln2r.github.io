Blade and Soul

List of commands related to Blade and Soul stuff. Including: daily and weekly challanges, dungeon guides, and item drop location.

Character Search - !who [character name]

Check character gear and stats.
Contains: health, attack power, defense, clan, faction, aliases, mystic, block, evasion, boss (ap-dp), critical hit, critical damage, weapon, gems, soulshields, accesories, HM points allocation, selected skills.

Available parameters:

  • [character name] Optional: character name.

Aliases: ps, f2
Example: !who wquin hollow

Smart Bid - !bid [player count] [item price/item name]

Get a calculation how much you should bid for an item. You can use item name if it's available.
Contains: max bid for personal use, max bid for market.

Available parameters:

  • [player count] Required: number of players present in party/raid.
  • [item price] Required*: bidded item price.
  • [item name] Required*: bidded item name.
*Optional if either item name or price used.

Aliases: smartbid
Example: !bid 6 200

Reaction Role

Add reaction role system by selecting existing message.

How To Use:

  • 1. Select existing message by copying the id (right click on the message, then copy id) by using !rmessage
  • 2. Add emoji and coresponding role to add when user select the reaction by using !radd or !raddonce, if you want to remove a reaction use !rremove

Selecting Message - !rmessage [message id]

To select existing message.

Available parameters:

  • [message id] Required: message id.

Example: !rmessage 728930197101084725

Adding Reaction - !radd [emoji] [role]

Adding reaction and the coresponding role.

Available parameters:

  • [emoji] Required: emoji.
  • [role] Required: mentioned role.

Example: !radd 💖 @helper

Adding Reaction for Single Use Only - !raddonce [emoji] [role]

Adding single use reaction and the coresponding role. (good for accepting rules)

Available parameters:

  • [emoji] Required: emoji.
  • [role] Required: mentioned role.

Example: !raddonce ✔ @member

Removing Reaction - !rremove [emoji]

To remove a reaction from the message.

Available parameters:

  • [emoji] Required: used emoji.

Example: !rremove 💖

New Member - !setting

Customizing "new member welcome system".

Selecting Channel

Add/Change: !setting welcome ch #channel-name
Disable: !setting welcome ch disable
Example: !setting welcome ch #welcome

Changing Welcome Message

Add/Change: !setting welcome msg message here
Disable: !setting welcome msg disable
Example: !setting welcome msg Hello! Welcome to our server!
Note: To add member name use MEMBER_NAME, to add server name use SERVER_NAME to add bot prefix use BOT_PREFIX.

Selecting Follow-up Channel (where member go after doing join commands)

Add/Change: !setting join ch #channel-name
Disable: !setting join ch disable
Example: !setting join ch #general

Selecting Member Role

Add/Change: !setting join role #role-name
Disable: !setting join role disable
Example: !setting join role @member

Join Message - !setting joinmsg [Your message here]

Change welcome message when user did !join command.
Note: If you want to mention the message author use MESSAGE_AUTHOR, and SERVER_NAME when you want to add the server name.

Add/Change: !setting join msg #channel-name
Disable: !setting join msg disable
Example: !setting join msg #news

Bot Admin Role - !setting admin [@role-name]

Selecting roles which can use administrative level commands.
Note: Guild/Server owner will always have permission regardles having the roles or not.

Add/Change: !setting admin @role-name
Disable: !setting admin disable
Example: !setting admin @admin
